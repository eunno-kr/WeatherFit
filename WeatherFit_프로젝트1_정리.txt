프로젝트1 - WeatherFit 프로젝트 정리
작성일: 2026-06-20

1. 프로젝트 개요
WeatherFit은 현재 날씨, 사용자 스타일 취향, 성별, 색감 선호, 개인 옷장 정보를 바탕으로 오늘 입을 옷을 추천하는 웹 서비스입니다.
단순히 날씨에 맞는 옷을 한 가지 제시하는 것이 아니라, 3가지 완성 코디와 개인 옷장 기반 조합을 함께 제공합니다.

2. 웹사이트 주요 타깃
- 날씨에 맞는 데일리 코디를 빠르게 결정하고 싶은 사용자
- 스타일, 색감, 성별 등 개인 취향을 반영한 추천을 원하는 사용자
- 본인이 가진 옷을 등록해 실제 옷장 안에서 조합을 받고 싶은 사용자
- 패션 용어에 익숙하지 않아 쉬운 설명이 함께 필요한 사용자

3. 핵심 기능

3-1. 사용자 스타일 프로필
- 나이대, 성별(남성/여성/공용), 위치, 선호 스타일 설정
- 온도 민감도(추위를 많이 탐/보통/더위를 많이 탐) 설정
- 색감 선호(무채색/밝은 톤/차분한 톤/포인트 컬러) 설정
- 주요 컬러 추천 선택 및 직접 컬러 입력

3-2. 실시간 날씨 기반 추천
- Open-Meteo 날씨 데이터를 사용해 현재 온도, 체감 온도, 습도, 최저/최고 온도, 일교차를 표시
- 비 여부와 체감 온도를 고려해 아우터, 상의, 하의, 신발, 포인트 아이템을 추천
- 날씨 요청이 실패할 경우 샘플 날씨 데이터를 표시해 화면이 비지 않도록 처리

3-3. 3가지 완성 코디 추천
- 미니멀, 캐주얼, 스트릿, 포멀, 스포티 스타일을 선택 가능
- 한 코디 안에는 아우터, 상의, 하의, 신발, 포인트 아이템을 각각 하나씩 제시
- 반바지, 긴바지 등 이해하기 쉬운 괄호 설명을 패션 용어와 함께 제공
- 각 아이템에 색상 칩을 표시하여 색상 조합을 직관적으로 확인 가능

3-4. 색상 조합 기능
- 사용자가 선택한 색감 선호를 반영한 팔레트 생성
- 추천 코디에는 팔레트의 주요 색상이 최소 한 번 이상 사용되도록 구성
- 밝은 톤을 선택했을 때 무채색만 반복되는 문제를 줄이도록 색상 기준을 분리
- 색상 칩은 추천 코디, 옷장 등록, 등록 옷 목록, 고정 아이템 기반 코디에 공통 적용

3-5. 개인 옷장 기능
- 옷 이름, 분류, 색상, 두께, 스타일, 비 오는 날 착용 가능 여부를 등록
- 기본 색상 팔레트, 확장 색상 팔레트, 직접 색상 선택기를 지원
- 등록된 옷 목록을 분류별로 필터링하고 삭제 가능
- 옷장 데이터는 브라우저 LocalStorage에 저장되어 새로고침 후에도 유지

3-6. 고정 아이템 기반 재조합
- 오늘 꼭 입고 싶은 아우터, 상의, 하의, 신발, 액세서리를 고정 가능
- 고정 아이템은 유지하고 나머지 부위는 날씨와 스타일에 맞는 등록 옷 또는 일반 추천으로 조합
- '고정한 옷으로 다시 조합하기' 버튼으로 결과를 갱신

4. 사용 기술
- React 19: 화면을 컴포넌트 단위로 구성하고 상태를 관리
- Vite: 빠른 개발 서버와 프로덕션 빌드 도구
- Tailwind CSS: 반응형 레이아웃과 스타일 구현
- JavaScript: 추천 규칙, 상태 처리, 데이터 저장 로직 구현
- Open-Meteo API: 날씨 데이터 조회
- LocalStorage: 프로필, 옷장, 고정 아이템 상태의 브라우저 저장
- GitHub Actions / GitHub Pages: 정적 웹사이트 자동 배포 설정
- Figma: 서비스 화면, 디자인 시스템, 사용자 흐름 시안 제작

5. 데이터 설계

사용자 프로필(Profile)
- age: 나이대
- gender: 성별
- cityName: 선택 도시
- style: 선호 스타일
- sensitivity: 온도 민감도
- colorTone: 색감 선호
- mainColor: 주요 컬러
- customColor: 직접 입력 색상

옷장 아이템(Wardrobe Item)
- id: 고유 식별값
- name: 옷 이름
- category: outer/top/bottom/shoes/accessory
- color: 색상 이름
- colorHex: 색상 칩 HEX 값
- warmth: 두께 또는 계절감
- style: 스타일 태그
- rainOk: 비 오는 날 착용 가능 여부

고정 아이템(Locks)
- outer, top, bottom, shoes, accessory 별로 사용자가 지정한 옷장 아이템 id 저장

6. 추천 작동 방식
1) 사용자가 프로필과 스타일을 설정한다.
2) 선택한 도시 기준으로 날씨 API를 호출한다.
3) 현재 온도, 체감 온도, 비 여부, 일교차를 계산한다.
4) 스타일, 성별, 온도 민감도, 색감 선호를 반영해 일반 추천 코디 3개를 만든다.
5) 옷장 아이템이 있으면 카테고리, 두께, 스타일, 색상, 비 착용 가능 여부를 비교해 옷장 기반 코디를 만든다.
6) 고정한 옷이 있으면 해당 옷을 우선 사용하고 나머지 부위를 조합한다.
7) 결과와 이유를 화면에 보여주고 데이터는 LocalStorage에 저장한다.

7. 화면 구성
- 프로필 설정 화면: 추천 정확도를 위한 기본 취향 입력
- 헤더/날씨 영역: WEATHERFIT 로고, 도시 선택, 프로필, 현재 날씨 리포트
- 스타일 선택 영역: 미니멀, 캐주얼, 스트릿, 포멀, 스포티 전환
- 메인 추천 영역: 오늘 입는 3가지 코디 및 추천 이유
- 색상 팔레트 영역: 코디에 사용되는 주요 색상 확인
- 내 옷장 영역: 옷 등록, 색상 선택, 목록 확인, 필터링, 삭제
- 고정 아이템 영역: 꼭 입고 싶은 옷을 기준으로 다시 조합
